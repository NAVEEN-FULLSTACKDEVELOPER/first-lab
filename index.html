  • 1. What is a programming language?
    Answer

    A programming language is a set of instructions that humans use to communicate with a computer to perform specific tasks.

  • 2. What is a machine-level language?
    Answer

    Machine-level language is a low-level language written in binary (0s and 1s), directly understandable by the computer processor.

  • 3. Give an example of a machine-level language.
    Answer

    Binary language (0 and 1).

  • 4. What is an assembly-level language?
    Answer

    Assembly-level language uses mnemonics (short words like ADD, SUB, MOV) instead of binary, but still requires an to translate it.

  • 5. What is a high-level language?
    Answer

    A high-level language is human-readable, easy to understand, and must be translated into machine code using a compiler or interpreter.

  • 6. Give two examples of high-level languages.
    Answer

    C, Java, Python.

  • 7. Who introduced the first machine-level language?
    Answer

    John von Neumann in 1940.

  • 8. Who introduced the first high-level language?
    Answer

    John Backus in 1950.

  • 9. What does an assembler do?
    Answer

    An assembler converts assembly language into machine code.

  • 10. What is a compiler?
    Answer

    A compiler translates an entire high-level program into machine code at once.

  • 11. What is an interpreter?
    Answer

    An interpreter translates and executes line by line instead of compiling the entire program at once.

  • 12. Example of a compiled language?
    Answer

    C, C++.

  • 13. Example of an interpreted language?
    Answer

    Python, JavaScript.

  • 14. What is a low-level language?
    Answer

    A low-level language is closer to machine code, harder for humans but faster for computers (e.g., machine language, assembly).

  • 15. What is a high-level language mainly used for?
    Answer

    To make programming easier for humans with readable syntax.

    Answer

    A procedural language follows step-by-step instructions (e.g., C).

    An object-oriented language is based on objects and classes (e.g., Java, C++).

  • 6. Define functional programming language with an example.
    Answer

    A functional language is based on mathematical functions (e.g., Haskell, Lisp).

  • 7. Which is faster: compiled or interpreted language? Why?
    Answer

    Compiled languages are faster because the code is fully converted before execution.

  • 8. Why is high-level language portable?
    Answer

  • 9. What is meant by platform independence?

    A language like Java can run on any OS because of the JVM (Java Virtual Machine).

  • 10. Which language type is closest to hardware?
    Answer

    Machine language (binary).

    Advantage: Fast execution.

  • 12. Mention one advantage and one disadvantage of high-level languages.
    Answer

    Advantage: Easy to learn.
    Disadvantage: Slower than low-level.

  • 13. What is the function of a linker?
    Answer

    A linker combines different program modules into a single executable file.

  • 14. Which programming paradigm does Python support?
    Answer

    Object-oriented, procedural, and functional.

  • 15. Why is assembly language faster than high-level languages?
    Answer

    • Hard Question

  • 1. Explain how source code becomes executable code.
    Answer

    The source code (written by programmer) → Compiler/Interpreter translates → Object code → Linker joins modules → Loader loads into memory → Executable program runs.

  • 2. Compare machine-level, assembly-level, and high-level languages with examples.
    Answer

    Machine-level: Binary (0s,1s) – Example: 1010.
    Assembly-level: Mnemonics (ADD, SUB) – Example: MOV AX, BX.
    High-level: Human-readable – Example: print("Hello").

  • 3. Why is Java called “Write once, run anywhere”?
    Answer

    Because Java code is compiled into bytecode, which runs on JVM across platforms.

  • 4. Differentiate between procedural and object-oriented programming.
    Answer

    Procedural: Based on functions and procedures (C).
    OOP: Based on objects and classes (Java).

  • 5. Explain the role of intermediate code in programming languages.
    Answer

    Some compilers (like Java) produce intermediate code (bytecode), which is later executed by a virtual machine.

  • 6. What is the difference between static typing and dynamic typing?
    Answer

    Static typing: Variables have fixed type at compile-time (C, Java).
    Dynamic typing: Types checked at runtime (Python, JavaScript).

  • 7. Why are low-level languages not portable?
    Answer

    Because they depend on hardware architecture, so the same code cannot run on different processors.

  • 8. What is the difference between compiler errors and runtime errors?
    Answer

    Compiler error: Detected during compilation (syntax errors).
    Runtime error: Detected during execution (divide by zero).

  • 9. What are scripting languages? Give two examples.
    Answer

    Scripting languages are interpreted languages used for automation (e.g., Python, JavaScript).

  • 10. How does a Just-In-Time (JIT) compiler improve performance?
    Answer

    JIT compiles bytecode into machine code at runtime, making execution faster.

  • 11. Differentiate between declarative and imperative programming.
    Answer

    Declarative: Focuses on what to do (SQL, Prolog).
    Imperative: Focuses on how to do (C, Java).

  • 12. What is the significance of mnemonics in assembly language?
    Answer

    Mnemonics make code easier to read by replacing binary with short words like ADD, SUB, MOV.

  • 13. Why is C called a middle-level language?
    Answer

    Because C supports both low-level (pointers, memory access) and high-level (functions, structured programming) features.

  • 14. What is the role of an IDE (Integrated Development Environment)?
    Answer

    An IDE provides tools like editor, compiler, debugger, and build system to help programmers.

  • 15. How is functional programming different from OOP?
    Answer

    Functional: Focuses on functions and immutability (Haskell).
    OOP: Focuses on objects and state (Java, Python).

    3. JVM loads bytecode.

    5. Program executes.
